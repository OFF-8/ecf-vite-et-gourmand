1. Réflexions technologiques initiales

BesoinChoixJustificationAlternative écartée
Application web dynamique + formulaires PHP 8 + HTML/CSS/JS Stack imposée / maîtrisée en formation DWWM ; rendu serveur simple à déployer et à expliquer. Node/React (surdimensionné pour le périmètre ECF).
UI responsive rapide Bootstrap 5 Grille, composants accessibles, gain de temps sur formulaires et navigation. CSS from scratch uniquement (trop long pour le planning).
Données métier relationnelles MySQL + PDO Intégrité (FK), transactions implicites via InnoDB, requêtes préparées (anti-injection). SQLite (moins adapté multi-utilisateurs / hébergement cloud).
Statistiques admin MongoDB (+ Chart.js) Exigence sujet NoSQL ; documents de stats flexibles, découplés du schéma relationnel. Uniquement SQL aggregations (ne satisfait pas l'exigence NoSQL).
Emails transactionnels PHPMailer (SMTP) Fiable en prod ; fallback mail() en local. mail() seul (souvent non fonctionnel en cloud).
Hébergement (sujet : Vercel / Fly / Azure / Heroku) Vercel (PHP) + MySQL Railway Déploiement rapide, HTTPS, CI via Git ; MySQL externalisé car Vercel n'héberge pas MySQL. AlwaysData (excellent PHP mais hors liste stricte du sujet déploiement cloud listé).

7. Sécurité — dispositions prises

RisqueMesure
Injection SQLRequêtes préparées PDO systématiques
XSShtmlspecialchars sur les sorties utilisateur
Mots de passepassword_hash / password_verify (bcrypt) ; règle de complexité à l'inscription
Fixation de sessionsession_regenerate_id(true) à la connexion
Contrôle d'accèsVérification du rôle (includes/auth-role.php) sur admin/employé
Création adminInterdite dans l'UI ; admin uniquement en fixtures SQL
Reset MDPToken unique + expiration en base
SecretsVariables d'environnement Vercel / config/local.php (gitignoré)
Messages d'authMessage d'erreur volontairement vague (pas d'énumération d'emails)
Install cloudRoute install.php protégée par clé

Limite connue (Vercel serverless) : les sessions PHP fichiers peuvent être instables entre lambdas froides. Acceptable pour la démo ECF ; en production durable → sessions Redis/DB.

8. Déploiement — démarche réalisée

  1. Préparer le runtime PHP sur Vercel (vercel.json, api/index.php, Composer/PHPMailer).
  2. Provisionner MySQL sur Railway (TCP Proxy public *.proxy.rlwy.net).
  3. Configurer les variables DB_* / DATABASE_URL sur Vercel (Production + Preview).
  4. Déployer (vercel --prod) → alias ecf-vite-et-gourmand-self.vercel.app.
  5. Exécuter une fois /install.php?key=vitegourmand2026 pour créer schéma + fixtures.
  6. Valider les parcours admin / employé / client.

Détails complémentaires : README.md, DEPLOIEMENT-ECF.md.
