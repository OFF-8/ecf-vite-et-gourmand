Documentation — Gestion de projet

Projet : Vite & Gourmand (ECF)
Méthode : approche agile légère (kanban) + Git Flow simplifié

1. Organisation choisie

Le projet a été découpé en fonctionnalités livrables (auth, menus, commande, espaces métier, déploiement). Chaque fonctionnalité correspond à une branche Git feature/* issue de develop.

  • Backlog : liste des user stories tirées du sujet ECF
  • En cours : une à deux features à la fois (limiter le WIP)
  • Revue / test : validation locale avant merge
  • Terminé : merge dans develop, puis release vers main

4. Règles d'équipe / qualité

  • Une branche = une fonctionnalité nommée clairement
  • Pas de push direct sur main sans passer par develop
  • Tester le parcours concerné avant merge
  • Ne jamais committer secrets (.env, local.php, mots de passe)
